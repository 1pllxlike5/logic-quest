# LogicQuest 线性代数学科集成方案

> 当前版本：v1.0 → v2.0（多学科支持）

---

## 一、变更概览

| 项目 | v1.0 现状 | v2.0 目标 |
|------|----------|----------|
| 学科 | 仅数字逻辑 | 数字逻辑 + 线性代数 |
| 登录后 | 直入主页 | 先选学科再入主页 |
| 题库 | 1 套 80 题 | 2 套共 ~170 题 |
| 数据隔离 | 按用户 | 按 `用户 + 学科` |
| 金币/等级 | 单一 | 跨学科互通 |
| 界面 | 固定标题 | 顶栏显示当前学科，可切换 |

## 三、数据结构变更

### 3.1 localStorage Key 变更

| v1.0 | v2.0 |
|------|------|
| `logicquest_accounts` | `logicquest_accounts` (不变) |
| `logicquest_{username}_v1` | `logicquest_{username}_{subjectId}_v1` |

如用户 `alice` 在数字逻辑：`logicquest_alice_digital_v1`  
如用户 `alice` 在线性代数：`logicquest_alice_linear_v1`

### 3.2 全局状态新增字段

```js
appState.currentSubject = "digital";  // "digital" | "linear"
appState.sharedCoins = 0;            // 跨学科金币（替换原 coins）
appState.sharedLevel = 0;            // 跨学科等级
```

## 四、线性代数题库结构

### 4.1 题型定义

| 题型 | type 值 | 数量 | 判题方式 |
|------|---------|------|---------|
| 选择题 | `choice` | 35 | 自动（选 index） |
| 判断题 | `tf` | 32 | 自动（选 true/false） |
| 计算题 | `calculation` | 18 | 自我评估（对/错按钮） |
| 综合题 | `comprehensive` | 4 | 自我评估 + 参考解答 |

### 4.2 数据格式

```js
// 选择题（同数字逻辑）
{
  id: 1001,
  type: "choice",
  subject: "linear",
  chapter: "行列式",
  difficulty: 2,
  question: "行列式 |1 0 1; 0 2 0; 1 0 3| 的值为（ ）。",
  options: ["2", "4", "6", "8"],
  answer: 1,
  explanation: "按第二行展开：2×(1×3-1×1)=2×2=4。"
}

// 判断题（新题型）
{
  id: 2001,
  type: "tf",
  subject: "linear",
  chapter: "行列式",
  difficulty: 1,
  question: "一个 n 阶行列式中某行元素与另一行元素的代数余子式对应相乘后的和必为零。",
  answer: true,
  explanation: "这是行列式展开定理的推论。"
}

// 计算题（类似现有 comprehensive）
{
  id: 3001,
  type: "calculation",
  subject: "linear",
  chapter: "行列式计算",
  difficulty: 3,
  question: "计算 n 阶行列式 |1+x 1 ... 1; 1 1+x ... 1; ...; 1 1 ... 1+x|",
  steps: ["每行减去第一行", "提取公因式", "化为上三角", "得结果"],
  answer: "x^(n-1) · (x+n)",
  explanation: "利用行列式的线性性质和降阶法。"
}

// 综合题
{
  id: 4001,
  type: "comprehensive",
  subject: "linear",
  chapter: "线性方程组",
  difficulty: 4,
  question: "已知三元非齐次线性方程组 AX=b 有特解 η1=(1,0,2)ᵀ, η2=(-1,2,1)ᵀ, η3=(1,0,0)ᵀ，R(A)=1，求通解。",
  steps: ["由解的结构知通解 = 特解 + 导出组通解", "导出组基础解系秩 = n-R(A) = 3-1 = 2", "取 η2-η1, η3-η1 为导出组基础解系", "通解 = η1 + k1(η2-η1) + k2(η3-η1)"],
  answer: "X = (1,0,2)ᵀ + k1(-2,2,-1)ᵀ + k2(0,0,-2)ᵀ, k1,k2∈R",
  explanation: "..."
}
```

### 5.3 题库数据嵌入方式

线性代数 89 题全部嵌入现有 `index.html`，作为第二个 `QUESTION_BANK`。两个题库共存于同一文件，根据 `currentSubject` 动态引用。

伪代码结构：
```js
const QUESTION_BANKS = {
  digital: [ ... 80 题 ... ],
  linear:  [ ... 89 题 ... ]
};

function getCurrentBank() {
  return QUESTION_BANKS[appState.currentSubject];
}
function getCurrentSubject() {
  return SUBJECTS[appState.currentSubject];
}
```

## 七、题型扩展

线性代数引入**判断题**（`tf` 类型），需要在刷题引擎中新增渲染：

```
┌──────────────────────────────────┐
│  判断题  |  ⭐ 难度 ★☆☆          │
│                                  │
│  一个 n 阶行列式中某行元素与     │
│  另一行元素的代数余子式对应相乘   │
│  后的和必为零。                  │
│                                  │
│   ┌──────────┐ ┌──────────┐     │
│   │  ✅ 正确  │ │  ❌ 错误  │     │
│   └──────────┘ └──────────┘     │
└──────────────────────────────────┘
```

- 两个按钮，点击即提交
- 自动判题：对比 `question.answer` (true/false)

## 九、注意事项

1. **题库质量**：PDF 提取的数学公式可能不完整，需人工校对关键公式
2. **文件大小**：89 题追加大约增加 50KB，总文件约 ~150KB，仍可接受
3. **旧数据兼容**：首次加载检测旧 key `logicquest_{user}_v1`，自动迁移到 `logicquest_{user}_digital_v1`
4. **后续扩展**：此架构支持继续追加学科（概率论、高数等），只需添加学科定义 + 题库
